<DocSection title="Overview">

The CLI does the repeatable parts around the fingerprint lifecycle: scaffold a
package, validate the node graph, compose context for a surface, select checks,
and emit advisory review packets. Your agent does the interpretation.

`ghost --help` shows the short core workflow. `ghost --help --all` shows the
complete command index, and `ghost <command> --help` shows flags for one
command.

The canonical fingerprint is a `.ghost/` directory tree of prose nodes:

```text
.ghost/
  manifest.yml          # schema + id
  index.md              # the core node, true everywhere (optional)
  <surface>/index.md    # a surface's own prose (the directory is the surface)
  <surface>/<node>.md   # a prose node placed in that surface
  checks/*.md           # optional ghost.check/v1 checks
```

The command tables below are generated from the CLI source. Run
`pnpm dump:cli-help` after command or flag changes.

#### Exit codes

Every command follows one contract, so an agent can branch on the exit code:

| Code | Meaning |
| --- | --- |
| `0` | Success. |
| `1` | The command ran but the result is unhappy: `validate` found issues, or an unexpected error was thrown. |
| `2` | The command was called wrong: a bad flag or argument, a node or surface that is not in the package (`ERR_UNKNOWN_SURFACE`), or a missing package for a command that needs one to do its work (`gather`, `checks`, `review`). |
| `3` | A command-specific refusal, currently only `skill install` when a skill is already present (pass `--force`). |

A missing package is reported, not treated as a crash: `validate` records it as
a finding (exit `1`).

</DocSection>

<DocSection title="Create And Inspect">

### Initialize: `init`

Scaffold a `.ghost/` package: a manifest and a core `index.md` node. Add
surfaces by adding directories (`checkout/index.md` is the `checkout` surface).
Use `--template <name>` to pick a starter, `--package <dir>` for an exact
directory, or set `GHOST_PACKAGE_DIR` when a host wrapper stores Ghost files
outside the default `.ghost`.

<CliHelp tool="ghost" command="init" hideDescription />

```bash
ghost init
ghost init --template default
ghost init --package product-surface
GHOST_PACKAGE_DIR=.agents/ghost ghost init
```

</DocSection>

<DocSection title="Validate">

### Validation: `validate`

Validate the package: artifact shape plus the node graph. Every `relates` link
resolves, there is exactly one root, and the graph is acyclic. Defaults to
`.ghost`; pass a file to validate a single node.

<CliHelp tool="ghost" command="validate" hideDescription />

```bash
ghost validate
ghost validate .ghost/checkout/trust.md
ghost validate --format json
```

</DocSection>

<DocSection title="Gather Context">

### Compose a surface slice: `gather`

With no argument, list every node by id and description so an agent can match a
task to one. With an exact node id, compose its context slice. An inexact query
(`ghost gather payment`) is not search: it returns the same node menu so the
agent re-picks by description, plus closest-id "did you mean" suggestions for a
likely typo, under the stable code `ERR_UNKNOWN_SURFACE`.

A slice composes three things:

- **full bodies along the path**: every file from the package root down to the
  surface's own folder. A sibling folder's nodes never appear.
- **edges** (full bodies, one hop): the `relates` targets of every node on that
  path, so a rule authored once high in the tree reaches every descendant. A
  link to a node also offers that node's subtree as pointers.
- **pointers**: the surface's own descendants and the subtree of any node it
  relates to, offered as `id` + `description` for the agent to pull on
  demand.

Use `--as` to filter full-body nodes to a single incarnation; untagged essence
nodes always pass.

<CliHelp tool="ghost" command="gather" hideDescription />

```bash
ghost gather
ghost gather checkout
ghost gather payment            # inexact: returns the menu + did-you-mean suggestions
ghost gather checkout --as email
ghost gather checkout --format json
```

This is the pre-generation step: Ghost gives agents surface-composition context
before they build, not only after a review finds drift.

Naming a surface that is not in the package is an error, not a silent empty
result: `gather`, `checks`, and `review` all emit the stable code
`ERR_UNKNOWN_SURFACE` with closest-id suggestions, so an agent can retry without
a human.

</DocSection>

<DocSection title="Govern Changes">

### Select checks: `checks`

List the markdown checks and ground the named surfaces. Every check is offered;
the agent names the surfaces the change touches to ground them, then judges
which checks apply. A check binds to the prose it enforces through its optional
`source:` pointer, not by surface routing. Use `--no-grounding` to omit the
grounded nodes and return only the checks.

<CliHelp tool="ghost" command="checks" hideDescription />

```bash
ghost checks --surface checkout
ghost checks --surface checkout,billing
ghost checks --surface checkout --format json
```

### Advisory review packet: `review`

Emit an advisory packet for a diff: touched surfaces, the offered checks, and
fingerprint grounding, with the diff embedded verbatim. Diff against a git ref
with `--base`, or pass a patch with `--diff` (use `-` for stdin).

<CliHelp tool="ghost" command="review" hideDescription />

```bash
ghost review --surface checkout --base main
ghost review --surface checkout --diff change.patch
git diff | ghost review --surface checkout --diff -
ghost review --surface checkout --format json
```

Wrappers should consume `--format json` and map Ghost severities (`critical`,
`serious`, `nit`) into their own review format. Advisory review is never a CI
gate on its own.

</DocSection>

<DocSection title="Skill And Maintenance">

### Install the skill: `skill`

Install the unified Ghost skill bundle so a host agent knows how to author and
use the fingerprint.

<CliHelp tool="ghost" command="skill" hideDescription />

```bash
ghost skill install
ghost skill install --agent claude
ghost skill install --dest ~/.codex/skills/ghost
```

### Migrate a legacy package: `migrate`

Migrate a legacy `.ghost/` package onto the node-graph surface model. Use
`--dry-run` to print the plan without writing.

<CliHelp tool="ghost" command="migrate" hideDescription />

```bash
ghost migrate
ghost migrate .ghost --dry-run
```

</DocSection>
