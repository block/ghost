# The `expression.md` format

A Ghost **expression** is a single Markdown file that captures what a design system is trying to say. It replaces the opaque `.ghost-fingerprint.json` with something humans can read and edit, and something LLMs can consume natively — while preserving a structured machine layer for `ghost compare`, `ghost review`, `ghost fleet`, and friends.

The file has two parts:

1. **Frontmatter (YAML)** — **the authoritative source of truth.** Every field the fingerprint carries lives here. Deterministic tools (`compare`, `review`, `fleet`, `viz`) read only this.
2. **Body (Markdown)** — a human-readable mirror of the narrative fields. Rendered for people and LLM prompts. **Never consulted by `parseExpression`.** If the body and frontmatter disagree, the frontmatter wins, silently.

Canonical filename: `expression.md` (flat, no dotfile, no slug prefix). Zero-config default for every Ghost command that reads a fingerprint.

Current schema version: **2**.

---

## Precedence rule (the one rule)

**Frontmatter is authoritative for every field.** The body is a readable mirror of Character, Signature, Decisions, and Values — regenerated by `ghost profile` alongside the frontmatter. It exists so the file reads like a design doc, but it is not merged into the fingerprint.

This replaces the earlier "body wins for narrative fields" contract (schema 1).

---

## Frontmatter schema

Validated by a zod schema (`packages/ghost-core/src/expression/schema.ts`) and published as JSON Schema at `schemas/expression.schema.json`. Below is the shape:

```yaml
---
# --- meta ---
name: Claude                      # display name
slug: claude                      # kebab-case id
schema: 2                         # format version — required, rejected on mismatch
generator: ghost@0.9.0            # tool + version that produced this file
generated: 2026-04-18T00:00:00Z   # ISO-8601 (alias for `timestamp`)
confidence: 0.87                  # 0–1, overall inference confidence (optional)
extends: ./parent.expression.md   # optional — inherit from a parent (see Composition)

# --- fingerprint: identity ---
id: claude
source: llm                       # registry | extraction | llm | unknown
timestamp: 2026-04-18T00:00:00Z
sources:                          # optional, lists the targets that were combined
  - github:anthropics/claude-code
  - https://claude.ai

# --- fingerprint: narrative ---
observation:
  summary: "A literary salon reimagined as a product page — warm, unhurried."
  personality: [restrained, editorial]
  distinctiveTraits:
    - "Warm ring-shadows instead of drop-shadows"
    - "Editorial serif/sans split"
  closestSystems: [notion, linear]

decisions:
  - dimension: warm-only-neutrals
    decision: "Every gray carries a yellow-brown undertone. No cool blue-grays."
    evidence: ["#5e5d59", "#87867f", "#4d4c48"]

values:
  do:
    - "Use Parchment (#f5f4ed) as the primary light background"
    - "Keep all neutrals warm-toned"
  dont:
    - "Use cool blue-grays anywhere"
    - "Mix sans-serif into headline slots"

# --- fingerprint: structured tokens ---
palette:
  dominant:
    - { role: accent, value: '#c96442' }
    - { role: surface, value: '#f5f4ed' }
  neutrals:
    steps: ['#faf9f5', '#e8e6dc', '#87867f', '#5e5d59', '#4d4c48', '#141413']
    count: 6
  semantic:
    - { role: error, value: '#b53333' }
    - { role: focus, value: '#3898ec' }
  saturationProfile: muted          # muted | vibrant | mixed
  contrast: moderate                # high | moderate | low

typography:
  families: ['Anthropic Serif', 'Anthropic Sans', 'Anthropic Mono']
  sizeRamp: [12, 14, 15, 16, 17, 20, 25.6, 32, 52, 64]
  weightDistribution: { 400: 0.6, 500: 0.4 }
  lineHeightPattern: loose          # tight | normal | loose

spacing:
  scale: [4, 8, 12, 16, 24, 32]
  baseUnit: 8                       # null if no coherent base
  regularity: 0.85                  # 0–1

surfaces:
  borderRadii: [8, 12, 16, 32]
  shadowComplexity: subtle          # none | subtle | layered
  borderUsage: moderate             # minimal | moderate | heavy

# --- fingerprint: vector layer ---
embedding: [0.12, -0.44, 0.33 /* …49 floats… */]
---
```

**Required:** `id`, `source`, `timestamp`, `palette`, `spacing`, `typography`, `surfaces`, `embedding`.
**Required-but-conditional:** `schema` (if present, must equal 2). Missing `schema:` is warned but accepted.
**Optional narrative:** `observation`, `decisions`, `values`. Omit rather than lie — a missing `observation` is truer than a fabricated one.
**Optional meta:** `name`, `slug`, `generator`, `confidence`, `generated`, `sources`, `extends`.

When `extends:` is present, required fingerprint fields may be omitted — the child inherits them from the parent. The merged result is re-validated against the strict schema.

## Body

The body is a readable mirror of the narrative fields. `ghost profile` emits it; `parseExpression` parses it into `ParsedExpression.body` for **diagnostic** use (lint, diff) but never merges it into the fingerprint.

Four section kinds, all optional, in this order:

- `# Character` — one paragraph, the atmosphere read (mirrors `observation.summary`)
- `# Signature` — 3–7 bullets, the distinctive moves (mirrors `observation.distinctiveTraits`)
- `# Decisions` — one `###` block per decision, with optional `**Evidence:**` (mirrors `decisions[]`)
- `# Values` — `## Do` / `## Don't` bullet lists (mirrors `values`)

If a section is missing from the body but present in the frontmatter, `ghost lint` will flag `body-sync` as a warning — the mirror is stale.

## Composition (`extends:`)

A child expression can inherit from a parent:

```yaml
---
schema: 2
extends: ./parent.expression.md
id: child-system
decisions:
  - dimension: warm-neutrals
    decision: "Now we also forbid warm grays."
    evidence: ["#3a3630"]
---
```

**Merge rules** (see `packages/ghost-core/src/expression/compose.ts`):

- **Scalars / arrays:** child replaces parent when present.
- **`decisions[]`:** merged by `dimension` — child wins per-dim; parent-only decisions preserved.
- **`palette.dominant` / `palette.semantic`:** merged by `role` — child wins per-role.

Cycles throw. Chains are resolved depth-first. After resolution, `extends:` is stripped from the returned meta.

Skip resolution: `loadExpression(path, { noExtends: true })`.

---

## Decision fragments

Large systems can split decisions across files. If a `decisions/` directory sits next to the expression.md, each `*.md` inside is read as a single decision and merged in by dimension:

```
my-system/
├── expression.md
└── decisions/
    ├── warm-neutrals.md
    ├── serif-headlines.md
    └── ring-shadows.md
```

Fragment format:

```markdown
---
dimension: warm-neutrals          # optional — falls back to filename stem
evidence: ['#5e5d59', '#87867f']  # optional
---

Every gray carries a yellow-brown undertone. No cool blue-grays exist anywhere.
```

Fragments override inline decisions with the same dimension. Skip with `loadExpression(path, { noFragments: true })`.

## Validation

`parseExpression` runs two gates on every read (unless `skipValidation: true`):

1. **Schema version gate.** `schema:` must equal 2. Stale files throw with a regenerate hint.
2. **Zod validation.** Structural errors are collected and surfaced with field paths:

   ```
   Invalid expression frontmatter:
     • palette.saturationProfile: Invalid enum value...
     • embedding.3: Expected number, received string
   ```

For tooling that wants to inspect partial or in-progress files, `skipValidation` bypasses both gates.

## Tooling surface

| Command | Does |
|---|---|
| `ghost profile . --emit` | Write `expression.md` (body mirror + frontmatter) |
| `ghost lint [path]` | Check schema validity, body/frontmatter sync, unused palette, broken evidence citations |
| `ghost expr-diff <a> <b>` | Semantic diff: decisions added/removed/modified, value deltas, palette role swaps, token changes |
| `ghost compare <a> <b>` | Vector distance (quantitative — use `expr-diff` for qualitative) |
| `ghost context [path]` | Emit a grounding skill bundle (`SKILL.md` + `expression.md` + `tokens.css`) |

Programmatic API (`@ghost/core`): `loadExpression`, `parseExpression`, `serializeExpression`, `lintExpression`, `compareExpressions`, `mergeExpression`, `loadDecisionFragments`, `FrontmatterSchema`, `toJsonSchema`.

## What's deliberately excluded

- **Body authority over frontmatter.** The body is a mirror, not a source. If you want to edit the fingerprint, edit the YAML. `ghost profile` regenerates the body.
- **Implementation-specific tokens.** No framework names, no CSS-in-JS specifics, no component library assumptions. Decisions are abstract ("warm-only neutrals"), not concrete ("`neutral-50` in `tailwind.config.js`").
- **Confidence theatre.** If the generator isn't sure, omit `confidence` or set `source: unknown`. Fabricated `1.0` is worse than missing.
- **Schema migration.** Schema 1 files are rejected outright. Regenerate.

## JSON Schema

`schemas/expression.schema.json` is regenerated from the zod source:

```bash
pnpm --filter @ghost/core build && node scripts/emit-expression-schema.mjs
```

Point your editor at it via a comment or `yaml.schemas` config for autocomplete in the frontmatter.
